# 我的初步想法

现在这个mcp服务的日志有两种：

- mcp命令操作日志，文件名格式为`YYYY-MM-DD_HH-MM-SS.log`
- 终端原始日志，文件名格式为 `sessionid_YYYY-MM-DD_HH-MM-SS.log`

### 1. mcp命令操作日志

文件名感觉有些长，直接去掉连字符，改为`YYYYMMDDHHMMSS.log`

### 2. 终端原始日志

终端原始日志只有sessionid，无法区分是哪个设备的，我希望可以加上设备名，例如`lubancat-sessionid-YYYYMMDDHHMMSS.log`

---

## 澄清后的最终方案

经过 spec 阶段的需求澄清，对最初想法做了三处调整（详见 spec.md）：

### 1. 时间戳格式：紧凑数字 → 日期分隔 + 时间紧凑

最初想法是去掉所有连字符（`YYYYMMDDHHMMSS`），但纯紧凑数字 `20260718135301` 一眼看不出日期/时间边界，与 sessionId 拼接后边界更模糊。

最终改为 **`YYYY-MM-DD_HHMMSS`**（折中）：
- 日期用 `-` 分隔，易读（`2026-07-18`）
- 时间紧凑无分隔符（`135301`）
- 时间戳与 sessionId 之间用 `_` 分隔，边界清晰
- 关键：只要各字段从大到小、定宽，字典序仍等于时间序，排序不受影响

### 2. 设备区分方式：加进文件名 → 按设备建子目录

最初想法是把设备名塞进文件名（`lubancat-sessionid-...log`）。

讨论中意识到：按设备名建子目录比塞进文件名更优——文件名保持简洁，目录结构本身就携带设备维度，且按设备聚合后查阅更方便。

最终方案：
```
.embedded/log/
├── 2026-07-18_135400.log              ← 命令日志，进程级，留在根目录
├── board-lubancat/                    ← 设备子目录（终端原始日志）
│   ├── serial_1_2026-07-18_135301.log
│   └── ssh_2_2026-07-18_140522.log
└── board-a/
    └── ...
```

### 3. 分目录范围：两类都分 → 只终端日志分

命令日志是进程级单例（一个进程只服务一个设备），作为「进程全局日志」留在 log 根目录，一眼能看出最近一次运行；只终端原始日志按设备分目录。

### 4. 设备名格式：剥 board- 前缀 → 用原值

最初想剥掉 `board-` 前缀用 `lubancat`，但 `board-a` → `a`、`board-b` → `b` 单字母辨识度差，且需要额外 strip 逻辑。

最终直接用 `resolveDeviceName()` 原值（如 `board-lubancat`），与 config.yaml / registry 完全一致，实现零额外逻辑。
